# Other deployment platforms

While we recommend Vercel, AudioGuideKit works on any platform that serves static files. Here's how to deploy to other popular options.

## Netlify

Netlify is very similar to Vercel—easy setup, free tier, automatic deploys.

### Deploy from GitHub

<Steps>
  <Step title="Sign up">
    Go to [netlify.com](https://netlify.com) and sign in with GitHub
  </Step>
  <Step title="Add New Site">
    Click "Add new site" → "Import an existing project"
  </Step>
  <Step title="Connect GitHub">
    Select your repository
  </Step>
  <Step title="Configure Build">
    - Build command: `npm run build`
    - Publish directory: `dist`
  </Step>
  <Step title="Deploy">
    Click "Deploy site"
  </Step>
</Steps>

### Custom domain on Netlify

1. Site settings → Domain management
2. Add custom domain
3. Configure DNS with Netlify's nameservers or CNAME records

### Netlify environment variables

Site settings → Environment variables → Add variable

## GitHub pages

GitHub Pages is free and works well for open source projects.

### Setup with GitHub actions

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Bun
        uses: oven-sh/setup-bun@v1

      - name: Install dependencies
        run: bun install

      - name: Build
        run: bun run build

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```

### Enable GitHub pages

1. Repository Settings → Pages
2. Source: "GitHub Actions"
3. Your site will be at `https://username.github.io/repo-name/`

### Base path configuration

If deploying to `username.github.io/repo-name/` (not root), update `vite.config.ts`:

```typescript
export default defineConfig({
  base: '/repo-name/',
  // ...
});
```

<Warning>
GitHub Pages serves from a subdirectory by default. If your repo is named `tour`, the URL will be `username.github.io/tour/`. Configure the base path accordingly.
</Warning>

## Cloudflare pages

Cloudflare Pages offers generous free limits and a global CDN.

### Deploy from GitHub

<Steps>
  <Step title="Sign up">
    Go to [pages.cloudflare.com](https://pages.cloudflare.com)
  </Step>
  <Step title="Create Project">
    Connect to Git → Select repository
  </Step>
  <Step title="Configure Build">
    - Framework preset: Vite
    - Build command: `npm run build`
    - Build output directory: `dist`
  </Step>
  <Step title="Deploy">
    Save and Deploy
  </Step>
</Steps>

### Cloudflare benefits

- Unlimited bandwidth (free tier)
- Automatic SSL
- 500 builds/month on free tier
- Fast global edge network

## AWS S3 + CloudFront

For complete control and scalability, use AWS.

### Prerequisites

- AWS account
- AWS CLI configured
- S3 bucket created
- CloudFront distribution (for CDN and HTTPS)

### Deploy script

Create `deploy.sh`:

```bash
#!/bin/bash

# Build
bun run build

# Sync to S3
aws s3 sync dist/ s3://your-bucket-name --delete

# Invalidate CloudFront cache
aws cloudfront create-invalidation \
  --distribution-id YOUR_DISTRIBUTION_ID \
  --paths "/*"

echo "Deployed!"
```

### S3 static website configuration

1. Enable static website hosting on the bucket
2. Set index document: `index.html`
3. Set error document: `index.html` (for SPA routing)

### CloudFront setup

1. Create distribution pointing to S3 bucket
2. Enable HTTPS with AWS Certificate Manager
3. Configure custom domain if desired

<Note>
AWS setup is more complex but gives you fine-grained control over caching, security, and scaling.
</Note>

## Firebase hosting

Good if you're already using Firebase for other services.

### Install Firebase CLI

```bash
npm install -g firebase-tools
firebase login
```

### Initialize

```bash
firebase init hosting

# Select options:
# - public directory: Dist
# - single-page app: Yes
# - GitHub actions: Optional
```

### Deploy

```bash
bun run build
firebase deploy
```

## Platform-agnostic tips

### Always build first

```bash
bun run build
```

Before deploying, make sure the build succeeds locally.

### Test the build

```bash
bun run preview
```

This serves the production build locally so you can test before deploying.

### HTTPS is required

PWA features (Service Worker, offline mode) require HTTPS. All platforms above provide free SSL. Don't use HTTP-only hosting.

### SPA routing

AudioGuideKit uses client-side routing. Configure your host to redirect all paths to `index.html`:

- **Vercel/Netlify**: Automatic
- **S3**: Set error document to `index.html`
- **Apache**: Use `.htaccess` with rewrite rules
- **Nginx**: Use `try_files $uri $uri/ /index.html`

### Caching headers

For optimal performance, configure caching:

- **HTML**: No cache (always fresh)
- **JS/CSS**: Long cache (hashed filenames)
- **Assets**: Long cache

Most platforms handle this automatically for Vite projects.
